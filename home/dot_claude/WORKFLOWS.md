# Workflows

## Feature Implementation

1. Analyze existing patterns
2. Write tests first (TDD)
3. Implement following patterns
4. Run tests and verify
5. Create PR

## Debugging

1. Read error and identify location
2. Investigate systematically
3. Propose minimal fix for root cause
4. Verify fix doesn't break tests

## Code Review

1. Check diff scope and purpose
2. Look for bugs, type safety, error handling
3. Verify tests present
4. Check performance and security

## Communication

- Be direct and concise
- Use bullet points
- Include code when relevant
- Explain only significant decisions
